# React Flow 性能优化方案

本文档总结了对流程图系统的性能优化方案，主要针对渲染性能和响应速度的提升。

## 1. 任务处理系统重构

### 优化前
- 复杂的任务队列系统，包含多种任务类型和优先级
- 任务超时保护和防抖机制增加了复杂性
- 临时节点和SSE消息处理混合在一起

### 优化后
- 简化为专注处理SSE消息的队列
- 移除不必要的任务优先级、超时保护和防抖
- 保留文本追加合并功能，提高文本更新效率
- 将taskQueue改为chatMessageQueue，更符合实际用途
- 临时节点操作直接处理，不经过队列，提高响应速度

## 2. 临时节点创建优化

### 优化前
- 临时节点创建有2秒延迟
- 等待DOM更新的setTimeout设置为200ms

### 优化后
- 减少等待时间，将setTimeout设置为50ms
- 直接更新状态，不使用动画以提高响应速度
- 在SSE请求处理时禁用用户输入，避免冲突

## 3. 节点组件渲染优化

### 优化前
- 节点组件频繁重新渲染
- 没有使用React.memo和useCallback优化

### 优化后
- 使用React.memo包装BaseNode组件，避免不必要的重新渲染
- 添加自定义比较函数，只在关键属性变化时重新渲染
- 使用React.memo包装其他节点组件（如QueryNode）
- 使用useCallback包装Flow组件中的回调函数

## 4. 节点高度计算优化

### 优化前
- 每次布局都重新计算节点高度
- 频繁的DOM操作影响性能

### 优化后
- 添加高度缓存机制，使用节点ID和内容哈希作为键
- 优先使用节点的height属性，避免重复计算
- 确保容器div只创建一次，减少DOM操作
- 提供缓存清理函数，便于管理内存

## 5. 其他优化

### 布局和动画优化
- 根据节点数量动态调整动画时长
- 当节点数量过多时自动跳过动画
- 减少不必要的布局计算

### 状态管理优化
- 添加isProcessingSSE状态，提供更好的用户体验
- 在SSE请求处理时禁用用户输入，避免冲突

## 后续优化建议

1. **虚拟化渲染**：对于大量节点的场景，考虑使用虚拟化渲染技术，只渲染可见区域的节点
2. **Web Worker**：将布局计算移至Web Worker中，避免阻塞主线程
3. **代码分割**：使用React.lazy和Suspense实现组件懒加载
4. **预渲染**：对于静态内容，考虑使用预渲染技术
5. **进一步优化样式**：减少CSS计算和重绘

## 性能监控

建议添加性能监控指标，如：
- 首次渲染时间
- 节点渲染时间
- 布局计算时间
- 用户交互响应时间

通过这些指标，可以持续评估和改进系统性能。
