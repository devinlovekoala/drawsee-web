# 流程图紧凑布局优化

## 优化内容

为了让流程图的节点排布更加紧凑，提高空间利用率，我们对布局算法进行了精细调整。

### 1. 节点间距优化

在 `entitreeFlexLayout` 函数中调整了关键间距参数：

#### 水平间距调整
- **firstDegreeSpacing**: `75 → 45` (-30像素)
  - 相同父节点的子节点之间的水平间距
  - 减少40%，使同级节点更加紧凑

- **secondDegreeSpacing**: `20 → 15` (-5像素)  
  - 不同父节点的节点间距
  - 进一步优化空间利用

- **nextAfterSpacing/nextBeforeSpacing**: `10 → 8` (-2像素)
  - 细微调整相邻节点间距

#### 垂直间距优化
- **sourceTargetSpacing**: `120 → 85` (-35像素)
  - 父子节点之间的垂直间距
  - 减少29%，显著提高垂直空间利用率

### 2. 智能缩放适配

针对更紧凑的布局，相应调整了智能缩放参数：

#### 基础缩放比例提升
```typescript
// 因为布局更紧凑，可以适当提高缩放比例
nodeCount <= 2:  1.2 → 1.4 (+0.2)
nodeCount <= 4:  1.0 → 1.2 (+0.2) 
nodeCount <= 8:  0.8 → 1.0 (+0.2)
nodeCount <= 15: 0.6 → 0.8 (+0.2)
nodeCount <= 25: 0.4 → 0.6 (+0.2)
nodeCount > 25:  0.25 → 0.4 (+0.15)
```

#### 视图模式优化
- **概览模式**：
  - maxZoom提高: `0.4/0.6/0.8 → 0.5/0.7/0.9`
  - minZoom提高: `0.05 → 0.08`
  - 更好地利用紧凑布局的优势

- **详情模式**：
  - maxZoom提高: `2.0 → 2.2`
  - minZoom提高: `0.8 → 1.0`
  - padding减少: `0.3 → 0.25`

### 3. 视觉效果改进

#### 空间利用率提升
- 水平方向节点间距减少约40%
- 垂直方向节点间距减少约29%
- 整体布局更加紧凑，同屏可显示更多内容

#### 可读性保持
- 保持足够的间距避免视觉拥挤
- 智能缩放自动调整到最佳查看比例
- 概览/详情模式快速切换

### 4. 性能影响

- **布局计算**: 参数调整不影响计算性能
- **渲染效果**: 更紧凑的布局减少了画布尺寸需求
- **用户体验**: 提高了信息密度，减少了滚动需求

## 预期效果

### ✅ 改进点

1. **更高的信息密度**
   - 同样的屏幕空间可以显示更多节点
   - 减少用户滚动和缩放操作

2. **更好的整体视图**
   - 复杂流程图的全局结构更容易理解
   - 节点关系更加紧密清晰

3. **保持可读性**
   - 虽然更紧凑，但仍保持良好的视觉层次
   - 智能缩放确保细节可见

### 📊 数值对比

| 参数 | 原值 | 新值 | 变化 |
|------|------|------|------|
| 同级节点间距 | 75px | 45px | -40% |
| 父子节点间距 | 120px | 85px | -29% |
| 初始缩放(≤4节点) | 1.0x | 1.2x | +20% |
| 概览模式最大缩放 | 0.6x | 0.7x | +17% |
| 详情模式最大缩放 | 2.0x | 2.2x | +10% |

### 🎯 使用场景优化

- **简单流程** (≤4节点): 更大的初始缩放，突出内容细节
- **中等复杂度** (5-15节点): 平衡的间距和缩放，最佳可读性
- **复杂流程** (15+节点): 紧凑布局提供更好的全局视图

通过这次优化，流程图的空间利用率得到显著提升，同时保持了良好的可读性和用户体验。用户现在可以在更小的空间内查看更多内容，减少了缩放和滚动的需求。
