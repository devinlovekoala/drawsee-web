# 流程图智能缩放优化方案

## 问题分析

原有的流程图存在用户体验问题：
1. **固定缩放比例**：所有流程图都使用50%的固定缩放，不够灵活
2. **缩放范围限制**：fitView的缩放范围过于严格（0.75-0.90），无法适应不同内容
3. **缺乏智能适配**：没有根据节点数量、内容复杂度和屏幕尺寸来调整视图

## 解决方案

### 1. 智能自适应缩放系统

创建了 `useAdaptiveZoom` Hook，实现：

#### 🎯 智能初始缩放
- **节点数量适配**：
  - ≤2个节点：1.2倍放大（突出细节）
  - 3-4个节点：1.0倍标准显示
  - 5-8个节点：0.8倍适中显示
  - 9-15个节点：0.6倍紧凑显示
  - 16-25个节点：0.4倍概览显示
  - >25个节点：0.25倍全局显示

- **屏幕尺寸适配**：根据当前屏幕与标准屏幕(1920x1080)的比例动态调整
- **合理范围限制**：自动计算maxZoom和minZoom的合理范围

#### 🔄 双重视图模式
- **概览模式**：优化全局视图，适合理解整体结构
  - 自动调整padding和缩放范围
  - 节点多时使用更小的padding，更好利用屏幕空间
  
- **详情模式**：聚焦节点细节，适合阅读具体内容
  - 支持更大的缩放比例（最高2.0倍）
  - 为详情节点优化显示参数

#### ⚡ 智能节点聚焦
- **类型感知缩放**：不同类型节点使用不同的缩放策略
  - 详情节点（answer-detail, circuit-detail等）：更大缩放支持
  - 普通节点：适中缩放
- **上下文保持**：聚焦单个节点时保持周围节点的可见性

### 2. 用户界面优化

在右侧工具栏新增：
- **概览按钮**：一键切换到概览模式
- **详情按钮**：一键切换到详情模式  
- **缩放控制**：手动放大/缩小按钮
- **保留原有功能**：重新布局、小地图、详情面板等

### 3. 系统参数优化

#### ReactFlow配置优化
```typescript
maxZoom: 3        // 支持更大缩放（原来2）
minZoom: 0.05     // 支持更小缩放（原来0.01）
defaultViewport: { zoom: 0.8 }  // 更合理的默认值（原来0.5）
```

#### fitView配置优化
```typescript
maxZoom: 1.5      // 更灵活的最大缩放（原来0.9）
minZoom: 0.3      // 更灵活的最小缩放（原来0.75）
padding: 0.2      // 更合理的边距（原来0.3）
```


### 关键函数

#### `calculateOptimalZoom(nodes, targetNodeType?)`
- 根据节点数量和类型计算最佳缩放参数
- 返回 `{ maxZoom, minZoom, padding }`

#### `smartFitView(nodeIds, nodes, delay?, duration?, targetNodeType?)`
- 智能执行fitView操作
- 自动选择最佳缩放参数

#### `switchToOverviewMode(nodes)` / `switchToDetailMode(nodes)`
- 快速切换视图模式
- 针对不同模式优化参数

## 用户体验改进

### ✅ 解决的问题

1. **自动适配**：流程图会根据内容复杂度自动选择合适的初始缩放
2. **灵活缩放**：支持更大的缩放范围，既能看到全局又能查看细节
3. **快速切换**：提供概览/详情模式的一键切换
4. **智能聚焦**：选中节点时使用最适合该节点类型的缩放参数

### 🎯 使用场景

- **简单流程**（1-4个节点）：自动放大显示，突出内容细节
- **中等复杂度**（5-15个节点）：平衡全局和细节，用户可快速切换模式
- **复杂流程**（15+个节点）：自动缩小显示全局，提供概览模式

### 📱 响应式适配

- 自动检测屏幕尺寸
- 小屏幕设备自动调整缩放比例
- 保持在所有设备上的良好体验

## 性能优化

- 延迟执行fitView避免渲染冲突
- 智能判断是否需要重新布局
