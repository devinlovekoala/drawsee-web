四层分工模型

Layer 1
React Flow 画布层（你已有）
负责拖拽、连线、元件显示 — 不参与计算
  • →从 React Flow 的 nodes + edges 中提取网表（Netlist）
  • →每次图变化时，触发 analyzeCircuit() 重新构建仿真矩阵
  • →仿真结果反写回 node 的 data 字段，驱动自定义渲染
Layer 2
MNA 仿真引擎（需自建 or 移植）
纯 JS 类，运行在主线程或 Web Worker
// 核心数据结构（对标 CircuitJS CirSim.java） class CircuitEngine { nodes: Map<string, number> // nodeId → 矩阵行号 elements: CircuitElement[] // 所有元件实例 matrix: number[][] // MNA G矩阵 rhs: number[] // 右侧向量 B solution: number[] // 求解结果：各节点电压 analyzeCircuit() // 构建节点编号映射 stampCircuit() // 每个元件写入矩阵 runCircuit() // 每帧：LU分解 → 求解 → doStep() }
关键洞察：CircuitJS 的秘密在于 线性电路只做一次 LU 分解（stampCircuit），之后每帧只更新右侧向量（O(n²) 而非 O(n³)），这就是为什么它能每帧运行。
Layer 3
requestAnimationFrame 驱动循环
仿真时钟，每帧推进物理时间
// 仿真主循环（对标 CirSim 的 updateCircuit()） function simLoop(timestamp) { const stepsPerFrame = 20 // 每帧跑 20 个小时间步 for (let i = 0; i < stepsPerFrame; i++) { engine.runCircuit() // 推进 dt = 5e-6 秒 } renderFrame() // 把仿真结果写入 React state requestAnimationFrame(simLoop) }
为什么每帧跑 20 步？电路时间步需要很小（微秒级）才稳定，但 RAF 是 16ms 一帧，所以需要在一帧内跑多个小时步，然后只渲染最后一个结果。
Layer 4
Canvas / SVG 覆盖层
实时绘制电压色彩 + 电流流动动画点
  • →在 React Flow 画布上叠加一个 绝对定位的 Canvas，与 Flow 坐标系同步
  • →导线颜色：绿色=正压、红色=负压、灰色=地，厚度 ∝ 电流大小
  • →移动黄点：offset += current × speed，用 stroke-dashoffset 动画实现

MNA 矩阵"盖印"机制（CircuitJS 原理直译）

CircuitJS 开源了其内部原理文档 INTERNALS.md，以下是核心数学映射到代码的方式。
电阻 stamp
Vb/R - Va/R = I，向矩阵写入 4 个点
// 节点 a=行1，节点 b=行2，R=1000Ω stampResistor(a, b, R) { const g = 1 / R // 电导 matrix[a][a] += g // G矩阵对角线 matrix[b][b] += g matrix[a][b] -= g // 非对角（负电导） matrix[b][a] -= g }
电压源 stamp
需要额外行/列来求解电流（MNA 扩展）
// 电压源：Va - Vb = Vs，新增一行 vsIndex stampVoltageSource(a, b, vsIndex, voltage) { stampVCVS(a, b, vsIndex, 1) // 添加约束方程 rhs[vsIndex] = voltage // 设置电压值 } // 求解后 solution[vsIndex] 即为电源电流
电容 stamp（时域）
用伴随电路模型，等效为电导 + 电流源
// 电容用后向欧拉离散化：等效 Geq = C/dt doStep(C, dt) { const geq = C / dt const ieq = geq * (volts[0] - volts[1]) // 上一步电压 stampResistor(a, b, 1/geq) stampCurrentSource(a, b, ieq) // 历史电流源 }

CircuitJS 可视化效果的精确复现方案

效果 1
导线颜色编码（电压热图）
绿=高压 / 灰=零 / 红=负压
// 每帧渲染：根据节点电压给导线上色 function getWireColor(voltage, maxV) { const v = Math.max(-1, Math.min(1, voltage / maxV)) if (v > 0) return `rgb(0, ${Math.round(v*200)}, 0)` // 绿 if (v < 0) return `rgb(${Math.round(-v*200)}, 0, 0)` // 红 return '#888' // 灰（地） } // 在叠加 Canvas 上沿 edge 路径绘制彩色线段
效果 2
流动黄点（电流动画）
CircuitJS 最标志性的视觉效果
// 每帧：累积 offset，让虚线沿导线方向流动 class WireRenderer { offset = 0 draw(ctx, path, current, dt) { this.offset += current * CURRENT_SPEED * dt this.offset %= DOT_SPACING // 循环 ctx.setLineDash([3, DOT_SPACING - 3]) ctx.lineDashOffset = -this.offset // 负号=正向流动 ctx.strokeStyle = '#FFD700' ctx.lineWidth = 2 ctx.stroke() } }
这个效果本质上是 CSS/Canvas 的 stroke-dashoffset 动画，不需要真的绘制每个点，非常高效。
效果 3
元件上的实时数值标注
悬浮/常驻显示电压、电流、功率
  • →把仿真结果写回 React Flow 的 node.data.simResult
  • →自定义 NodeComponent 读取 simResult 渲染覆盖标签
  • →Hover 时用 Tooltip 显示完整参数（V/I/P/频率）
// 自定义 React Flow 节点组件 const ResistorNode = ({ data }) => ( <div className="resistor-node"> <ResistorSymbol /> {data.simResult && ( <div className="sim-overlay"> {formatValue(data.simResult.voltage)}V {formatValue(data.simResult.current)}A </div> )} </div> )
效果 4
画布内示波器（Scope）
点击导线/节点，弹出实时波形
  • →仿真引擎维护每个被探测节点的 ringBuffer（环形缓冲区，存 1000 个历史点）
  • →用 Canvas 2D 绘制波形，每帧增量绘制新数据（而非重绘全部）
  • →支持多通道（多个节点同时探测，不同颜色区分）

推荐实施顺序（最小可验证路径）

1
搭建 MNA 引擎骨架 1~2周
仅支持 R、V源、I源。目标：一个 RC 串联电路能跑起来，能在控制台输出节点电压。不做任何可视化。先验证数学是对的。
参考：直接读 CircuitJS 的 INTERNALS.md，或参考 npm 包 circuit-simulator 的 MNA 实现。
2
接入 RAF 主循环 + 基础可视化 1周
把 MNA 引擎接入 requestAnimationFrame。在 React Flow 画布上叠加 Canvas，实现导线颜色编码和流动点动画。此时用户已经能感受到"活的电路"了。
3
扩展元件 + Web Worker 卸载 2周
加入 C、L（时域离散），二极管（Newton-Raphson迭代）。把仿真引擎移入 Web Worker，通过 SharedArrayBuffer 或 postMessage 传递结果，保证主线程（UI）不卡。
4
示波器 + 交互体验打磨 1~2周
实现 Scope 组件（环形缓冲区+Canvas绘图）。滚轮调参数、拖动滑块实时看电路响应。Ngspice 后端保留作为"高精度验证"模式，两者共存。
最需要避开的坑：不要一开始就尝试复制 CircuitJS 的全部元件库。先把 R-L-C + 电压源 + 二极管 这 5 种做扎实，覆盖了 80% 的经典模拟电路，然后再扩展。数学不对的引擎加再多元件也没用。
可以直接复用的开源 JS MNA 实现：
• jsCircuit — 轻量，适合理解原理后移植关键部分
• CircuitJS1 源码的 Java → 人工翻译为 JS（GWT 编译产物可读性差，建议手翻核心 stamp 方法）
• analog-circuit-simulator npm 包 — 有 MNA + LU 分解现成实现
