# 昭析 · 电路可视化工具开发指导
---

## 一、定位与设计原则

**功能位置：** 替换左侧边栏原"AI 动画生成"入口，新名称建议为 **"电路可视化"** 或 **"交互图解"**。点击后在主内容区渲染生成器库页面，与对话树完全独立，无会话绑定。

**核心原则：** 纯前端，无后端调用，无 AI 介入。为本科模电/数电典型知识点各自制作一个参数化交互组件，聚合成生成器库。学生/老师调参，图形实时响应，全部在浏览器本地计算完成。

## 六、P0 核心场景实现规范

### 6.1 BJT Q 点与负载线

**计算 Hook**

```typescript
// useLoadLine.ts
export function useLoadLine(p: BJTParams): LoadLineResult {
  return useMemo(() => {
    const Vb  = p.Vcc * p.Rb2 / (p.Rb1 + p.Rb2);
    const Ic  = Math.max(0, (Vb - p.Vbe) / p.Re);
    const Vce = p.Vcc - Ic * (p.Rc + p.Re);

    const region =
      Ic === 0   ? 'cutoff' :
      Vce < 0.2  ? 'saturation' : 'active';

    const Ic_max = p.Vcc / (p.Rc + p.Re);
    const rbe    = 200 + 101 * 0.026 / Math.max(Ic, 1e-6);
    const Au     = region === 'active' ? -(p.beta * p.Rc) / rbe : 0;

    // 输出特性曲线族（8条，理想模型）
    const curves = Array.from({ length: 8 }, (_, i) => {
      const Ib  = (i + 1) * Ic_max / p.beta / 5;
      const Ica = p.beta * Ib;
      const pts = Array.from({ length: 201 }, (_, j) => {
        const vce = p.Vcc * j / 200;
        return [vce, vce < 0.2 ? Ica * vce / 0.2 : Ica] as [number, number];
      });
      return { Ib, pts };
    });

    return {
      qPoint:  { Ic, Vce, region },
      loadLine: [[0, Ic_max], [p.Vcc, 0]] as [number,number][],
      curves,
      xMax: p.Vcc,
      yMax: Ic_max * 1.25,
      rbe, Au,
    };
  }, [p]);
}
```

**D3 图表** (`Chart.tsx`)

- 坐标轴：x = VCE (V)，y = IC (mA)，深色网格线
- 曲线族：暗灰 `#334155`，细线
- 负载线：`#38bdf8` 蓝色虚线
- Q 点：颜色随工作区变化（放大区绿、饱和区橙、截止区灰）+ 光晕 + 坐标投影虚线 + 文字标注
- 每次 params 变化：`useEffect` 中 `selectAll('*').remove()` 全量重绘

**UI** (`index.tsx`)

- 三个滑块：RB1（10k–100kΩ）、RB2（2k–30kΩ）、RC（0.5k–10kΩ）
- 四格数值卡片：VCE / IC / rbe / |Au|
- 工作区状态 badge（右上角）
- 固定参数说明（Vcc=12V · Re=1kΩ · β=100）

### 6.3 滤波器幅频特性（接口规范）

```typescript
// 支持：一阶低通/高通、二阶低通/高通（Sallen-Key 模型）
// 可调：fc（对数滑块，10Hz–100kHz）、Q（0.3–3，二阶时显示）、类型选择
// 计算：500 个对数均匀采样点
//   一阶低通：|H| = 1 / √(1 + (f/fc)²)
//   二阶低通：|H| = 1 / √((1-(f/fc)²)² + (f/fc/Q)²)
// D3：x 对数轴（d3.scaleLog），y 线性 dB 轴，标注 -3dB 点与斜率
```

### 6.4 三种组态增益对比（接口规范）

```typescript
// 可调：β（20–300）、RC（kΩ）、RE（kΩ）、负载 RL（kΩ）
// 计算（小信号模型，固定 Ic=1mA 用于 rbe）：
//   共射 Au = -β·RC / rbe
//   共基 Au ≈  β·RC / rbe（同相，幅值近似相同）
//   共集 Au ≈  (1+β)RE / (rbe + (1+β)RE)  ≈ 1
// D3：三根竖条（柱状图）实时更新，y 轴为增益幅值，共射加反相标注
```

## 七、通用开发规范

**D3 与 React 共存：** 所有 D3 操作在 `useEffect` + `useRef` 内，每次重绘前 `selectAll('*').remove()`，不做增量更新。

**滑块性能：** `onChange` 套 `requestAnimationFrame` 节流，每帧最多触发一次 state 更新。

**滑块对数缩放：** fc 等跨越量级的参数，滑块值取 log，显示时还原：
```typescript
// 对数滑块：min=1, max=5 → 实际值 = 10^sliderVal
const fc = Math.pow(10, sliderVal);
```

**文件长度：** 每文件严格不超过 200 行，计算/渲染/UI 三个文件分离。

**Tailwind 版本：** 锁定 3.4.13，不使用 4.x 特性。

**路由：** 在项目路由配置中将原"AI 动画生成"路由指向 `VisualizerPage`，路径沿用或更新为 `/visualizer`。
