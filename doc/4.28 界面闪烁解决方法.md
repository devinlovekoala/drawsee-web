# 界面闪烁问题解决方案

## 问题描述
在使用鼠标滚轮进行缩放时，界面出现闪烁现象，这是由于频繁的布局重新计算以及节点内容重复渲染导致的。

## 解决方案
### A. 布局重新计算优化
优化了 `flow.tsx` 中的节点变化处理逻辑，主要改动如下：

#### 1. 新增变化追踪机制
```typescript
// 节点高度变化的累计值
let totalHeightChange = 0;
let changedNodesCount = 0;
let significantChanges = 0;
```

#### 2. 精确的变化检测算法
```typescript
if (curHeight !== undefined && newHeight !== undefined) {
  const heightDiff = Math.abs(newHeight - curHeight);
  totalHeightChange += heightDiff;
  changedNodesCount++;

  // 检查是否是显著变化
  if (heightDiff > 20) { // 显著变化阈值
    significantChanges++;
  }
}
```

#### 3. 智能布局重新计算触发条件
```typescript
// 计算是否需要重新布局的条件
const averageHeightChange = changedNodesCount > 0 ? totalHeightChange / changedNodesCount : 0;
const significantChangeRatio = significantChanges / totalNodes;

// 判断是否需要重新布局
needsRelayout = (
  // 条件1: 显著变化的节点比例超过阈值
  significantChangeRatio > 0.3 ||
  // 条件2: 平均高度变化超过阈值
  (changedNodesCount > 0 && averageHeightChange > 30) ||
  // 条件3: 变化的节点数量超过总节点数的一定比例，且平均变化较大
  (changedNodesCount > totalNodes * 0.4 && averageHeightChange > 15)
);
```

### B. 节点预览显示优化
优化了 `BaseNode.tsx` 中的节点预览逻辑，改进了缩放时的内容显示：

#### 1. 简化预览文本提取
```typescript
function extractPreview(text?: string): string {
  if (!text) return '';
  
  // 移除所有LaTeX公式和代码块
  const cleanText = text
    .replace(/\$\$([\s\S]*?)\$\$/g, '')
    .replace(/\$(.*?)\$/g, '')
    .replace(/```[\s\S]*?```/g, '')
    .replace(/#{1,6}\s+/g, '')      // 移除标题标记
    .replace(/\*\*/g, '')           // 移除粗体标记
    .replace(/\*/g, '')             // 移除斜体标记
    .trim();

  // 匹配第一句话（以。！？.!?结尾）
  const match = cleanText.match(/^[^。！？.!?]+[。！？.!?]/);
  if (match) {
    const firstSentence = match[0].trim();
    // 限制预览文本长度为15个字
    return firstSentence.length > 15 ? firstSentence.substring(0, 15) + '...' : firstSentence;
  }
  
  return cleanText.length > 15 ? cleanText.substring(0, 15) + '...' : cleanText;
}
```

## 优化效果

1. **减少重新布局频率**：
   - 只有在真正需要时才触发布局重新计算
   - 通过多个条件综合判断，避免不必要的重新布局

2. **提升性能**：
   - 减少了不必要的布局计算
   - 降低了 CPU 和内存使用
   - 优化了节点预览内容的渲染逻辑

3. **更好的用户体验**：
   - 减少界面闪烁
   - 保持流畅的缩放体验
   - 更清晰的预览内容显示

## 具体阈值说明

1. **显著变化判定**：
   - 单个节点高度变化超过 20px 被视为显著变化
   - 显著变化节点比例超过 30% 时触发重新布局

2. **平均变化判定**：
   - 平均高度变化超过 30px 时触发重新布局
   - 变化节点数量超过 40% 且平均变化超过 15px 时触发重新布局

3. **预览文本限制**：
   - 预览文本最多显示15个字
   - 优先显示完整的第一句话
   - 超出长度限制时添加省略号
